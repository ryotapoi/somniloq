# Step 1-a: UX シナリオ

**CLI の出力やインタラクションに関わる変更の場合は必須。** 内部ロジックのみの変更（ユーザー向け出力の変更なし）の場合は「N/A — ユーザー向け変更なし」と明記してスキップ。

以下の形式でユーザーに提示する:

```
## UX Scenario

### Before
- 具体的な CLI 入出力の状態を記述（どのコマンドで何がどう出るか）

### After
- 同じ操作がどう変わるべきかを記述

### 操作手順
1. ユーザーがどのコマンドを実行したら
2. 出力がどう変わるか
```

ポイント:
- 抽象的な仕様ではなく、**具体的な1つの状態**で書く
- 操作の前後で出力がどう見えるかを明示する
- 複数のシナリオがある場合は主要なものを2-3個

提示後、「UX シナリオを確認してください」と明示的に伝え、承認を待つ。承認された UX シナリオは Step 1-c のプランに含める。
