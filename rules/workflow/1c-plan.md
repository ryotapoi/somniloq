# Step 1-c: プラン作成・レビュー

## プラン作成

Plan モードでなければ EnterPlanMode する。調査・設計の結果をプランとして整理する。

## プランレビュー

プランの記述が完了したら、ExitPlanMode より先に必ず `/review-plan-all` スキルを Skill ツールで実行する。レビューを通るまで修正する。

レビュー完了後に ExitPlanMode を呼ぶ。

### レビュー指摘への対応

レビュー指摘に対応しない場合、プランを書き直して**考慮したこと**（不要と判断した理由・別タスクに切り出す理由・トレードオフの認識など）が読み取れる形にする。

- 完了宣言（「対処済み」「明記済み」など）だけで済ませず、事実と理由で書く
- 「考慮したこと」は対応する論点の近くに残す。プラン本文を読めば、なぜその論点を採用しなかったかが分かる状態にする

### レビューの収束条件

レビュー指摘が返ってきたとき、以下のどちらかを満たせば収束（次フェーズに進む）。

- 指摘が 0 件
- 残った指摘すべてについて、指摘の根拠（A）が前回の指摘と同じ
  - 「指摘の文面」ではなく「**根拠（why）が同じか**」で判定する
  - 同じ根拠 A の再指摘 = main 側で考慮を書いた論点に対し、レビューが同じ角度から再度指摘している状態。新しい角度ではないので収束扱い
  - 根拠が異なる A' であれば、別の角度として新たに考慮を書き、プランを更新してレビューに戻る

3 回目のレビューでも指摘が残っていて、収束扱いにできるか判断に迷うときはユーザーに「続行するか / さらに修正するか」を確認する。
