# Step 2b: 動作確認

実装が完了したら、CLI の挙動を実際に動かして確認する。somniloq は CLI ツールなので、`go test` だけでは「ユーザーから見た出力・対話」をカバーしきれない。

確認の観点:

- **CLI 出力**: 新規・変更したコマンドの stdout / stderr が想定通りか
- **対話的な挙動**: フラグの組み合わせ、エラー時の終了コード、help 文言の整合
- **副作用**: 実 DB ファイル・JSONL の読み書きが意図通りか（テスト用 DB を作って確認）

実行手段:

- 自分で Bash から `bin/somniloq <args>` を叩き、出力を読み取る
- 再現が難しい / 環境依存がある場合は、ユーザーにコマンド実行と出力共有を依頼する

次（レビュー）に進むのは挙動が想定通りだと確認してから。
