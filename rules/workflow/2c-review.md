# Step 2c: レビュー

動作確認を通過したら、`/review-code-all` を必ず実行する。コミット前のレビューは必須。

- `/review-code-all` は実装レビューの全チェーンを起動する。レビューはこのチェーンスキル経由で実行する（個別の `/review-code`, `/review-code-somniloq`, `/review-code-codex` 等は呼ばない）
- レビュー指摘があれば対応し、**再度 `/review-code-all` を通す**。コミットへ進むのはレビューを通してから

## レビューの収束条件

レビュー指摘が返ってきたとき、以下のどちらかを満たせば収束（次フェーズに進む）。

- 指摘が 0 件
- 残った指摘すべてについて、指摘の根拠（A）が前回の指摘と同じ
  - 「指摘の文面」ではなく「**根拠（why）が同じか**」で判定する
  - 同じ根拠 A の再指摘 = main 側で対応方針を整理した論点に対し、レビューが同じ角度から再度指摘している状態。新しい角度ではないので収束扱い
  - 根拠が異なる A' であれば、別の角度として新たに対応し、実装を更新してレビューに戻る

3 回目のレビューでも指摘が残っていて、収束扱いにできるか判断に迷うときはユーザーに「続行するか / さらに修正するか」を確認する。
