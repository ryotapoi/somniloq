# 001: 実装言語に Go を採用

- 日付: 2026-03-28
- ステータス: 採用

## 文脈

cclog は JSONL パース + SQLite 書き込みがメインの CLI ツール。
候補として Go, Swift, Rust, Python を検討した。

## 決定

Go を採用する。

## 理由

- **JSON の柔軟性**: 未知フィールドはデフォルトで無視される。スキーマ変更に強い
- **シングルバイナリ**: `go install` で PATH に入る。依存なし
- **起動速度**: CLI として毎回呼ばれるため重要
- **SQLite**: `modernc.org/sqlite` で CGO 不要の pure Go 実装が使える
- **CLI ツールの実績**: Go は CLI ツールの定番言語

## 却下した選択肢

- **Swift**: SQLite ライブラリ（GRDB）は良いが SPM ビルドが重く、CLI の起動も Go より遅い
- **Rust**: 型安全性は最強だが、このツールの規模に対してビルド時間と記述量がオーバーキル
- **Python**: 起動が遅く、依存管理が煩雑。型がランタイムで効かず、壊れてることに気づきにくい
