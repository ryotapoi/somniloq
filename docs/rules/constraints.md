# 技術的制約・禁止事項

- 後方互換性のためだけの shim / deprecated / fallback 分岐を追加しない
- `--no-verify` でフックをスキップしない
- 明示的な指示なしに force push しない

## SQL とデータの意味

- 文字列カラムの `MAX()` で「最新」「代表」を選ばない。例外は RFC3339 UTC のタイムスタンプ列（辞書順 = 時刻順が成立する）だけで、それ以外の TEXT カラムでは辞書順比較になり意味が壊れる。レビューポイント: `MAX(<TEXT カラム>)` を含む SQL 変更では対象カラムの順序の意味を確認する
- 集約クエリで GROUP BY キーと表示値に別々の短縮・変換を適用しない（情報が縮退して一覧に同名行が並ぶ）。レビューポイント: 集約の変更では GROUP BY キーと出力列の変換が一致しているか確認する

## 取り込みと保存済みデータ

- import のフィルタ・スキップ・バリデーションを変更したら、既存 DB に保存済みのデータとの食い違いを確認する（新規取り込み分と過去データで内容が変わる。backfill や `--full` 再取り込みの案内が必要かも含めて判断する）
- 取り込み日時などの「現在時刻」は処理単位で 1 回取得して共有する。レコード処理ループの中で都度取得しない（`internal/core/import.go` のテスト差し替え可能な `timeNow` を使う）

## CLI 表面

- フラグ・サブコマンド・モードを追加・変更したら、`cmd/somniloq` の usage 定数・ヘルプ文字列に同じコミットで反映する（CLI 仕様の正本 `docs/rules/scope.md` と README の同期は scope.md 冒頭の規定に従う）
